## קולקשנים ממשרד החינוך (נטענים מ-CSV)

כל המסמכים בקולקשנים אלו כוללים שדות metadata קבועים:

| שדה | סוג | תיאור |
|-----|-----|--------|
| `_id` | ObjectId | מזהה MongoDB אוטומטי |
| `source_file` | string | שם קובץ ה-CSV המקורי (דוגמה: `12345_03_2025CHESHBONIT.csv`) |
| `import_timestamp` | Date | תאריך ושעת הייבוא |
| `run_id` | string | מזהה ריצת הייבוא (פורמט: `RUN_1234567890`) |
| `file_type` | string | סוג הקובץ (CHESHBONIT / MUCARIM / וכו') |

### CHESHBONIT — חשבוניות תשלום

קולקשן ראשי. כל שורה מייצגת חשבונית תשלום של מוסד לנושא מסוים.

| שדה | סוג | תיאור |
|-----|-----|--------|
| `קוד_נושא` | number | קוד הנושא התשלומי |
| `תאור_נושא` | string | שם הנושא |
| `חודש_חישוב` | Date | חודש החישוב (Date object, UTC, יום ראשון בחודש) |
| `חודש_תחולה` | Date | חודש התחולה (Date object) |
| `סמל_מוסד` | number | סמל המוסד |
| `שם_מוסד` | string | שם המוסד |
| `יתרת_ביצוע_החודש` | number | **סכום לתשלום** — זהו הסכום הקובע בחישוב |
| + שדות נוספים | - | שדות נוספים מהקובץ המקורי |

**שימוש בעיבוד**: מסכמים `יתרת_ביצוע_החודש` לפי `קוד_נושא` ומשווים לפירוט.

## קולקשנים לניהול המערכת

---

### noseme — נושאים תשלומיים

מאגר כל הנושאים שהמערכת יכולה לעבד. **חיוני לעיבוד** — כל נושא שמופיע ב-CHESHBONIT חייב להיות כאן.

| שדה | סוג | תיאור |
|-----|-----|--------|
| `_id` | ObjectId | מזהה MongoDB |
| `code` | string | **קוד הנושא** (ייחודי, indexed) |
| `name` | string | שם הנושא |
| `table_type` | string | סוג הטבלה: `'מוסדות'` או `'ידנים'` |
| `seif_zhut` | string | **סעיף זכות** — קוד סעיף חשבונאי (מצד הזכות) |
| `seif_hova` | string | **סעיף חובה** — קוד סעיף חשבונאי (מצד החובה) |

**אינדקס**: `{ code: 1 }` unique

**upsert logic**: `updateOne({ code }, { $set: doc }, { upsert: true })`

### mosdot — מוסדות חינוך

מאגר מוסדות החינוך בנתיבות. נטען מקובץ מקור: `MOSDOT`.

| שדה | סוג | תיאור |
|-----|-----|--------|
| `_id` | ObjectId | מזהה MongoDB |
| `code` | string | **סמל מוסד** (ייחודי, indexed) — מקביל ל-`סמל_מוסד` בטבלות המקור |
| `name` | string | שם המוסד |
| `nihul_atsmi` | number | **סעיף לניהול עצמי** — מאוחזר כש-`noseme.seif_hova/zhut = "ניהול עצמי"` |
| `hazana` | number | **סעיף להזנה** — מאוחזר כש-`noseme.seif_hova/zhut = "הזנה"` |
| `krav` | number | **סעיף לקרב** — מאוחזר כש-`noseme.seif_hova/zhut = "קרב"` |
| `sachar` | number | **סעיף לשכר** — מאוחזר כש-`noseme.seif_hova/zhut = "שכר"` |

**אינדקס**: `{ code: 1 }` unique

**upsert logic**: `updateOne({ code }, { $set: doc }, { upsert: true })`

## קשרים בין קולקשנים

```
CHESHBONIT.קוד_נושא  ←→  noseme.code          (חובה לקיום — כל נושא ב-CHESHBONIT חייב ב-noseme)
MUCARIM.קוד_נושא     ←→  noseme.code          (לבניית פקודות — קביעת סוג הנושא וסעיף תקציבי)
SHARATIM.קוד_נושא    ←→  noseme.code          (לבניית פקודות — קביעת סוג הנושא וסעיף תקציבי)
YADANIIM.קוד_נושא    ←→  noseme.code          (לבניית פקודות — קביעת סוג הנושא וסעיף תקציבי)
MUCARIM.סמל_מוסד     ←→  mosdot.code          (חיוני — לאיתור הסעיף התקציבי כשהנושא מכיל טקסט)
SHARATIM.סמל_מוסד    ←→  mosdot.code          (חיוני — לאיתור הסעיף התקציבי כשהנושא מכיל טקסט)
YADANIIM.סמל_מוסד    ←→  mosdot.code          (חיוני — לאיתור הסעיף התקציבי כשהנושא מכיל טקסט)
COMMANDS.חודש_חישוב  ←→  CHESHBONIT.חודש_חישוב (מפתח מחיקה)
runs.run_id          ←→  מסמכי runs           (לוג)
```

### זרימת קביעת הסעיף התקציבי

כאשר `noseme.seif_hova` או `noseme.seif_zhut` מכיל **טקסט** (לא מספר), הסעיף נקבע לפי:

```
שורה (MUCARIM/SHARATIM/YADANIIM)
    ↓ קוד_נושא
noseme.seif_hova / noseme.seif_zhut  ← מכיל טקסט (למשל: "שכר")
    ↓ מיפוי: "שכר" → שדה `sachar` במוסד
שורה.סמל_מוסד
    ↓
mosdot.sachar  ← הערך כאן הוא הסעיף התקציבי
    ↓
seif_hova = mosdot.sachar  (אם הטקסט היה ב-seif_hova)
seif_zhut = mosdot.sachar  (אם הטקסט היה ב-seif_zhut)
```

**לכן**: לכל שורה בנתוני הפירוט, הסעיף התקציבי הסופי נקבע מהשילוב של:
- `קוד_נושא` → `noseme` → סוג הסעיף + האם ישיר או חיפוש
- `סמל_מוסד` → `mosdot` → ערך הסעיף (כשהנושא מכיל טקסט)

## הערות חשובות

1. **Date fields** — כל שדות התאריך נשמרים כ-`Date` object ב-MongoDB (לא string)
2. **String vs Number** — `קוד_נושא` מגיע מה-CSV כ-number, אך ב-noseme נשמר כ-string. תמיד `String(code)` לפני השוואה
3. **Sparse data** — לא כל הקולקשנים מכילים נתונים לכל חודש
4. **No schema validation** — MongoDB ללא schema validation, כל הוולידציה ב-application layer
5. **Indexes** — לא הוגדרו indexes ידניים מלבד ה-`_id` האוטומטי. מומלץ להוסיף index על `חודש_חישוב` בקולקשנים ההיסטוריים
