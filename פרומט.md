# פרומט לבניית מערכת ניהול פקודות תשלום - עיריית נתיבות

## הנחיות לבינה מלאכותית

בנה לי אפליקציית Web מלאה מקצה לקצה לניהול פקודות תשלום עבור עיריית נתיבות.

---

## סקירה כללית

המערכת מאפשרת לעובדי הכספים בעירייה לייבא קבצי CSV ממשרד החינוך, לעבד אותם לפקודות חשבונאיות, ולהפיק דוחות. כל הנתונים נשמרים ב-MongoDB.

---

## סטק טכנולוגי

- **Frontend + Backend**: Next.js 15+ עם App Router, TypeScript strict
- **Database**: MongoDB (ללא Mongoose — משתמשים ישירות בדרייבר הרשמי `mongodb`)
- **Styling**: TailwindCSS, עיצוב כהה בסגנון GitHub Dark, RTL עברית מלאה
- **ייצוא**: ספריית `xlsx` לייצוא Excel
- **CSV parsing**: ספריית `csv-parser` עם טיפול ב-BOM (Byte Order Mark)
- **שפה**: עברית בלבד בממשק המשתמש, RTL (`dir="rtl"`)

## חיבור MongoDB (`lib/mongo.ts`)

```typescript
import { MongoClient, Db } from 'mongodb';

const MONGO_URI = process.env.MONGODB_URI!;
const DB_NAME = process.env.MONGODB_DB || 'pkuda_db';

const globalWithMongo = global as typeof globalThis & { _mongoClient?: MongoClient };

export async function getDb(): Promise<Db> {
  if (!globalWithMongo._mongoClient) {
    globalWithMongo._mongoClient = new MongoClient(MONGO_URI);
    await globalWithMongo._mongoClient.connect();
  }
  return globalWithMongo._mongoClient.db(DB_NAME);
}
```

**משתני סביבה נדרשים (`.env.local`)**:
```
MONGODB_URI=mongodb://localhost:27017
MONGODB_DB=pkuda_db
```

## דף העלאת קבצים (`/upload`)

### פורמט שם קובץ CSV
```
{מספר}_{חודש}_{שנה}{FILETYPE}.csv
דוגמה: 12345_03_2025CHESHBONIT.csv
```
- מספר: מזהה הרצה (כל ספרות)
- חודש: 1-12
- שנה: 4 ספרות
- FILETYPE: אחד מ-CHESHBONIT, MUCARIM, SHARATIM, YADANIIM, GY, HASAOT, HASNET, HASMASLULIM, MISROT, MISROTGY, MOADON, MUTAVIM, SHEFI

### אפשרויות ייבוא כפולות
- **דלג** (ברירת מחדל) — אם הקובץ כבר קיים, דלג
- **דרוס** — מחק ייבא מחדש

### לוגיקת ייבוא CSV
1. בדוק אם `source_file` קיים במונגו → דלג/דרוס
2. קרא CSV עם `csv-parser`, `mapHeaders` להסרת BOM וניקוי רווחים
3. המר עמודות `חודש_חישוב` ו-`חודש_תחולה` לאובייקטי `Date` (פורמט: `MM/YYYY`)
4. המר ערכים מספריים מ-string ל-number
5. הוסף metadata: `source_file`, `import_timestamp`, `run_id`, `file_type`
6. שמור ב-MongoDB בקולקשן המתאים לפי FILETYPE

**API**: `POST /api/import` (multipart/form-data)
- שדות: `files[]`, `duplicateStrategy` ('skip'|'replace')
- מחזיר: `{ ok, runId, summary: [{ filename, collection, rows, status }] }`

## דף הכנת פקודה (`/prepare`)

### שלב 1 — בחירת חודש חישוב וחודש פיצול
- בחר **חודש חישוב** (month picker) — רק נתונים עם `חודש_חישוב` זה ייכללו
- בחר **חודש פיצול** (month picker) — מחלק פקודות לשתי תקופות:
  - `חודש_תחולה > חודש_פיצול` → **תקופה ראשונה**
  - `חודש_תחולה <= חודש_פיצול` → **תקופה שנייה**

### שלב 2 — תצוגה מקדימה
לפני ההרצה, האפליקציה מציגה תצוגה מקדימה של הנתונים הקיימים במונגו לחודש שנבחר:
- CHESHBONIT — חשבוניות לחישוב (עמודה: `יתרת_ביצוע_החודש`)
- MUCARIM — מכורים (עמודה: `הפרש_מחושב`)
- SHARATIM — שרתים (עמודה: `הפרש_מחושב`)
- YADANIIM — ידנים (עמודה: `סכום_מחושב`)

לכל קולקשן — tab עם טבלה, סה"כ שורות, ייצוא Excel.

**API**: `GET /api/mongo/months` — מחזיר חודשים זמינים

### שלב 3 — הרצת עיבוד
לחיצה על "הפק פקודה" → `POST /api/process` עם `{ calc_month, split_month }`

### תוצאות העיבוד — 6 טאבים
1. **סיכום** — מספר חשבוניות, פקודות, סה"כ כספי, שגיאות, אזהרות
2. **פקודות תקופה ראשונה** — `חודש_תחולה > חודש_פיצול`
3. **פקודות תקופה שנייה** — `חודש_תחולה <= חודש_פיצול`
4. **לוג שגיאות ואזהרות** — רשימת בעיות
5. **השוואה** — invoiceTotal מול baseTotal מול yadaniTotal לכל נושא
6. **לא נקלטו** — שורות שנדחו עם סיבה

## API הכנת פקודה (`POST /api/process`)

### קלט
```json
{ "calc_month": "2025-01", "split_month": "2025-04" }
```

### לוגיקת העיבוד (שלב אחר שלב)

1. **טעינת נתונים** — במקביל (Promise.all):
   - `noseme` — כל הנושאים (lookup map: `code → נושא`)
   - `mosdot` — כל המוסדות (lookup map: `code → מוסד`)
   - `CHESHBONIT` — חשבוניות לפי `חודש_חישוב`
   - `MUCARIM` — מכורים לפי `חודש_חישוב`
   - `SHARATIM` — שרתים לפי `חודש_חישוב`
   - `YADANIIM` — ידנים לפי `חודש_חישוב`

2. **לכל קוד נושא** מ-CHESHBONIT:
   a. בדוק שהנושא קיים ב-NOSEME → אחרת שגיאה + "לא נקלט"
   b. **בדוק `table_type`**:

   **אם `שונות`**:
   - קבץ ישירות מ-CHESHBONIT לפי `קוד_נושא + חודש_תחולה`
   - קבע סעיף (ישיר — מספר בנושא)
   - צור שורת פקודה לכל חודש תחולה

   **אם `מוסדות`**:
   - חשב `invoiceTotal` = sum(CHESHBONIT.יתרת_ביצוע_החודש) לנושא זה
   - חשב `baseTotal` = sum(MUCARIM.הפרש_מחושב) + sum(SHARATIM.הפרש_מחושב)
   - חשב `yadaniTotal` = sum(YADANIIM.סכום_מחושב)
   - בדוק התאמה:
     - `invoiceTotal = baseTotal` → מקורות: MUCARIM + SHARATIM
     - `invoiceTotal = baseTotal + yadaniTotal` → מקורות: MUCARIM + SHARATIM + YADANIIM
     - אחרת → שגיאה + "לא נקלט", עצור
   - קבץ לפי `קוד_נושא + קוד_מוסד + חודש_תחולה` (איחוד מקורות לשורה אחת)
   - קבע סעיף לכל שורה:
     - אם `seif_hova`/`seif_zhut` הוא מספר → סעיף ישיר
     - אם טקסט → חפש בשדה המתאים ברשומת המוסד
     - חוסר/שגיאה → דחה שורה

   c. **פיצול לתקופות** (לכל שורה):
      - `חודש_תחולה > חודש_פיצול` → **תקופה ראשונה**
      - `חודש_תחולה <= חודש_פיצול` → **תקופה שנייה**

3. **שמירה**:
   - מחק `COMMANDS` לפי `חודש_חישוב` → שמור את כל הפקודות החדשות
   - שמור `runs` עם מטאדאטה מלאה

### מיפוי טקסט סעיף לשדה במוסד
| טקסט בנושא | שדה במוסד |

| ניהול עצמי | `nihul_atsmi` |

## הערות חשובות למימוש

1. **RTL מלא** — כל הדפים עם `dir="rtl"`, טקסט עברי, `text-right` בטבלאות
2. **BOM בקבצי CSV** — הסר `\uFEFF` מתחילת שמות עמודות: `col.replace(/^\uFEFF/, '').trim()`
3. **פורמט תאריך CSV** — `MM/YYYY` (לא ISO). פרסור: חלוק ב-`/`, בדוק `p1 <= 12 && p2 > 1900`, צור `Date.UTC(year, month-1, 1)`
4. **שמות עמודות** — רווחים מומרים ל-underscore: `.replace(/\s+/g, '_')`
5. **ייצוא Excel** — תמיד כלול `ws['!views'] = [{ RTL: true }]`
6. **Connection pooling** — שמור את ה-MongoClient ב-global variable למניעת reconnect בכל request
7. **TypeScript strict** — אין `any`, טיפוס `unknown` ב-catch blocks, ממשקים לכל מבנה נתונים
8. **שגיאות** — כל API route מחזיר `{ ok: false, error: string }` עם status 500 בשגיאה
9. **סה"כ שורות** — בצד שמאל, סה"כ כסף בצד ימין (RTL logic)
10. **חישוב סוף חודש** — `new Date(year, month + 1, 0)` נותן את יום האחרון של החודש
11. **`table_type` קובע הכל** — בתהליך הפקודה, תמיד בדוק את `table_type` של הנושא לפני כל לוגיקה
12. **סעיף ישיר vs טקסט** — `seif_hova`/`seif_zhut`: אם מספר = ישיר. אם טקסט = חיפוש בשדה המתאים ב-MOSDOT
13. **כיוון הפיצול לתקופות**: `חודש_תחולה > חודש_פיצול` → תקופה **ראשונה**; `<= חודש_פיצול` → תקופה **שנייה**
14. **שלוש בדיקות התאמה** לנושאי מוסדות: מלאה (base בלבד) → עם ידניים → שגיאה
15. **קיבוץ שונות**: לפי `נושא + חודש_תחולה` בלבד, ללא מוסד. **קיבוץ מוסדות**: לפי `נושא + מוסד + חודש_תחולה`
